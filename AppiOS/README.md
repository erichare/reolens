# Reolens for iPad and iPhone

Companion iOS/iPadOS app target for the Reolens repo. Same brand and
streaming stack as the macOS app, with UI rebuilt around touch
paradigms (TabView on iPhone, three-column `NavigationSplitView` on
iPad).

## Generating the Xcode project

The `.xcodeproj` is xcodegen-generated from [`project.yml`](project.yml) —
only the spec is committed. Regenerate after editing the spec:

```sh
cd AppiOS
xcodegen generate
```

If you don't have xcodegen yet:

```sh
brew install xcodegen
```

## Building

```sh
xcodebuild \
  -project AppiOS/ReolensiOS.xcodeproj \
  -scheme ReolensiOS \
  -destination 'generic/platform=iOS Simulator' \
  CODE_SIGNING_ALLOWED=NO \
  build
```

To run on a real device, open `ReolensiOS.xcodeproj` in Xcode, pick
your team under Signing & Capabilities, and Run.

## What's wired up now (v0.5.0)

Feature parity with the macOS app is the explicit goal (AGENTS.md §1).
Almost everything in macOS is here on iPad and iPhone too. Carve-outs
that stay platform-specific are called out below.

### Cross-platform parity

- ✅ Live multi-camera grids — adaptive, spotlight, 2×2, 3×3, 4×4, 5×5,
  single. Long-press a tile to enter reorder mode (iOS home-screen
  jiggle); drag to rearrange. Stills / Live toggle above the grid.
- ✅ Camera-list search (`.searchable`) and Save Snapshot to Photos
  via `.addOnly` PhotoKit permission.
- ✅ Per-channel detail with three tabs (Live / Recordings /
  Settings), PTZ, two-way talkback, rotate-per-stream.
- ✅ Recording timeline phase one — day-density calendar above the
  recordings list + per-day segment strip with AI event markers.
  Tap a segment to play.
- ✅ AI event filter chips on recordings + per-AI-tag notification
  toggles (Person / Vehicle / Pet / Package / Face / Visitor).
- ✅ Continuity / Handoff — start a camera on iPhone, hand off to
  Mac via the system Handoff icon, and vice versa.
- ✅ iCloud Keychain Sync opt-in (Settings → "iCloud Keychain Sync"
  section). Off by default per AGENTS.md §4.
- ✅ Per-channel sidebar rows on iPad (mirrors macOS) — each Hub /
  NVR expands into its individual cameras via DisclosureGroup.
- ✅ Shortcuts / Siri — `OpenCameraIntent` works from the system
  prompt and inside Shortcuts apps.
- ✅ Local-network discovery on add-camera (iOS 0.4.1).
  CameraDiscovery's Bonjour + HTTP /24 sweep is now wired into
  `AddCameraView` so users don't have to type the IP.
- ✅ TOFU TLS pinning for HTTPS cameras with self-signed certs
  (0.4.1, all platforms).
- ✅ Notification tap → exact clip routing (0.4.1). Tapping a
  recording-aged motion notification opens the camera's
  Recordings tab and auto-plays the closest clip.
- ✅ Liquid Glass adoption (0.5.0) — toolbars, sidebars, tile
  chrome, popovers, sheets, chips, and HUDs all use the iOS 26
  `.glassEffect()` material via `ReolensGlass` design tokens.
- ✅ Clip bookmarks + MP4 export (0.5.0) — long-press a recording
  to bookmark; Bookmarks sheet lists / plays / exports trimmed
  MP4s via `ClipExporter`. Bookmark references sync via iCloud
  Drive.
- ✅ Motion privacy zones (0.5.0) — visual rectangle editor in
  per-channel Settings, writes back via `SetMask` with graceful
  firmware-not-supported fallback.
- ✅ Overnight digest (0.5.0) — daily local notification at the
  configured hour (default 07:00), `DigestDetailView` sheet with
  24-hour bar chart, paired with `MotionDigestWidget` on the
  Home Screen.
- ✅ Custom recording scrubber (0.5.0) — thumbnail rail beneath
  the AVPlayer with content-addressed `ThumbnailCache` (500 MB
  LRU).

### iOS / iPadOS-only

- ✅ Picture-in-Picture (iOS 15+) — keep a camera in a floating
  window while using other apps. No macOS analog; documented carve-
  out per AGENTS.md §1.
- ✅ Pinch-to-zoom + drag-to-pan on the single-channel view (up to
  4×, double-tap to reset). macOS uses window-resize + Mission
  Control instead.
- ✅ Audio background mode for talkback + PiP.
- ✅ **Live Activities + Dynamic Island (0.5.0)** — in-flight
  motion-event activities with compact / expanded / minimal
  presentations. Replace-on-new-event (no stacking) with a 4 h
  auto-dismiss cap. Trigger-frame thumbnails live in the App
  Group container and are purged at the cap.
- ✅ **Lock Screen widgets (0.5.0)** — `LastMotionWidget` ships
  in the accessory families (inline / circular / rectangular)
  alongside its Home Screen siblings. macOS gets `systemSmall` /
  `systemMedium` variants instead.
- ✅ **Control Center widget (0.5.0)** — `OpenCameraControlWidget`
  binds `OpenCameraIntent` to a one-tap Control Center toggle
  on iOS 26+.
- ✅ **Stage Manager / multi-window (0.5.0)** — long-press any
  sidebar device / channel row for "Open in New Window".
  Powered by `WindowGroup(for: ReolensScene.self)` in
  `ReolensiOSApp.swift`. macOS gets the same via the matching
  context menu in `App/Views/CameraListView.swift`.

### Carve-outs (macOS-only)

- macOS Sparkle auto-update — App Store handles iOS updates.
- macOS menu-bar mode + Launch at Login — keeps the desktop app
  running with the window closed so motion notifications keep
  firing. iOS background notifications are delivered via the
  CloudKit relay (see below) when an always-on Mac on the same
  iCloud account is running Reolens in menu-bar mode.

## Engineering layout

- SPM libraries (`ReolinkAPI`, `ReolinkStreaming`, `ReolinkBaichuan`,
  `AppShared`) consumed via `..` package path. All build for iOS 26.
- Size-class-adaptive shell (iPhone tab bar vs iPad three-column
  NavigationSplitView).
- Camera list + grid prefs pulled from `CameraStore`, iCloud-synced.
- Local Network usage description for the LAN scan; microphone for
  talkback; `NSUserActivityTypes` for Continuity / Handoff.
- Audio background mode for talkback + PiP.
- **WidgetKit + ActivityKit extension** (`AppiOS/Widgets/`) shares
  the `group.com.reolens.Reolens` App Group with the main app.
  Extension entitlements grant **App Group filesystem only** — no
  `com.apple.security.network.client`. Data flow: main app writes
  via `SharedContainer.*`; extensions read the same files.
  AGENTS.md §16.
- **Live Activities** are managed by `MotionEventActivityController`
  in `AppiOS/Sources/LiveActivities/`. `EventNotifier.notify(...)`
  hands off to the controller via the `MotionEventLiveActivityBridge`
  protocol so `AppShared` doesn't import `ActivityKit`. The iOS app
  registers the concrete `MotionEventActivityBridge` at launch.
- **Multi-window scenes** (`ReolensScene` + `WindowGroup(for:)`) are
  routed through `CameraSceneHostiOS` in `ReolensiOSApp.swift`.
