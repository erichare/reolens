Reolens is a native client for Reolink cameras, NVRs, and Home Hubs, available
on macOS, iPad, and iPhone. SwiftUI, Swift 6 strict concurrency,
AVFoundation/VideoToolbox ‚Äî no Electron, no Java, no QtWebEngine. Cold
launches in under a second; battery-friendly; feels like every other native
Apple app on each platform.

**As of v0.3.0**, the macOS, iPadOS, and iPhone apps are at feature parity:
the same grid presets, tap-and-hold rearrange (with iOS-home-screen
jiggle), camera-list search, save-snapshot, and Shortcuts/Siri
integration on every platform ‚Äî plus Picture-in-Picture and digital
pinch-to-zoom on iOS/iPadOS. Your camera list, layouts, rotations, and
codec preferences sync via iCloud Drive. Passwords stay per-device in
Keychain ‚Äî they never leave the device they were entered on.

See [`AGENTS.md`](AGENTS.md) for the engineering principles that gate
every change (platform parity by default, no telemetry, no credentials
in logs) and [`AppiOS/README.md`](AppiOS/README.md) for the iOS-specific
project layout.

## Features

- **Cross-platform feature parity** ‚Äî every layout, gesture, and
  surface available on macOS is available on iPadOS and iOS (and vice
  versa) ‚Äî see [`AGENTS.md`](AGENTS.md) ¬ß1.
- **Live multi-camera grids** ‚Äî adaptive layout fills the window;
  switch to spotlight, 2√ó2, 3√ó3, 4√ó4, or 5√ó5 with one click ‚Äî on every
  platform.
- **Tap-and-hold rearrange** ‚Äî long-press any tile to enter
  iOS-home-screen-style jiggle mode; drag to reorder. Works in the
  channel grid AND the top-level device sidebar.
- **Camera search** ‚Äî `.searchable` field on every platform; case- and
  diacritic-insensitive match on display name or host. Useful when an
  NVR has 16+ channels.
- **Save Snapshot** ‚Äî capture the current frame on any tile. macOS
  writes a PNG to `~/Pictures/Reolens/` and reveals in Finder; iOS
  writes to Photos with the minimum `.addOnly` PhotoKit permission.
- **Picture-in-Picture (iOS/iPadOS)** ‚Äî keep a camera in a floating
  window while using other apps.
- **Pinch-to-zoom + drag-to-pan (iOS/iPadOS)** ‚Äî up to 4√ó digital zoom
  for inspecting detail. Double-tap to reset.
- **Shortcuts & Siri** ‚Äî "Hey Siri, open the Front Door camera in
  Reolens" works on every platform. The intent only stores a camera
  UUID ‚Äî no credentials cross the intent boundary.
- **Full PTZ** ‚Äî all 17 PTZ ops (pan, tilt, zoom, focus, presets,
  patrols) from the dedicated control bar.
- **Rich alarm notifications** ‚Äî when motion fires, get a notification
  with the trigger frame, not just text.
- **Two-way talkback** ‚Äî for Reolink cameras that support it.
- **Native streaming** ‚Äî RTSP / FLV / JPEG fallback, hardware-decoded
  with VideoToolbox.
- **Auto-updates (macOS)** ‚Äî Sparkle in-app updates from a single
  signed appcast. iOS goes through TestFlight / App Store.
- **iCloud sync** ‚Äî camera list, grid layout, channel order, rotations
  syncs across all your Apple devices. Passwords stay per-device in
  Keychain (never synced).
- **Open source** ‚Äî MIT-licensed, build it yourself if you prefer.

## System requirements

**macOS app**
- macOS 14 Sonoma or later
- Apple Silicon (M-series) or Intel ‚Äî universal binary

**iPad / iPhone app**
- iPadOS 18 / iOS 18 or later
- Any device that runs them

**Both**
- Reolink camera, NVR, or Home Hub on the local network
- HTTP/HTTPS access to the device's CGI port (default 80 / 443)

## Quick start

1. **Launch Reolens.** It'll ask for Local Network permission (needed to
   reach your cameras) and Notification permission (for motion alerts).
2. **Click the + in the sidebar** to add a camera. Enter the IP address
   (or hostname), username, and password ‚Äî the rest is auto-detected.
3. **Pick a camera in the sidebar** to view it. Use the layout picker
   (toolbar) to switch between adaptive / spotlight / 2√ó2 / 3√ó3 / 4√ó4 grids.

Drag tiles to rearrange them. Right-click a tile for "Make primary",
"Rotate", and per-channel settings.

`ReolinkAPI` knows nothing about SwiftUI, AppKit, or video frameworks ‚Äî it's
testable in isolation and ships as a standalone SPM library.

### Concurrency model

- `CGIClient` is an **actor** ‚Äî one instance per camera/NVR. Reolink
  devices have a notoriously small global session cap, so the actor
  serializes login/refresh and reuses one token across all commands.
- `CameraSession` is `@MainActor`-isolated and `@Observable` ‚Äî SwiftUI
  views observe `status`, `deviceInfo`, `channels`, `motionState`,
  `aiTriggered` directly.
- Token refresh is implicit; if the device returns `loginRequired`
  mid-session the client drops the token and retries once.

## API coverage today

Implemented as typed `Commands.*` constructors:

- `Login` / `Logout`
- `GetDevInfo`
- `GetAbility` (with capability-tree navigation + per-channel accessors)
- `GetChannelstatus`
- `GetMdState`
- `GetAiState` (people / vehicle / dog_cat / face / package / other / visitor)
- `PtzCtrl` (all 17 ops including presets, patrol, zoom, focus)
- `GetLocalLink`, `GetTime`, `GetOsd`, `SetOsd`, `GetHddInfo`
- `Search` (recordings list + per-day status)

Adding a new command is mechanical ‚Äî define a typed param + value model,
construct a `CGICommand`, and call `client.send(_:as:)`. See
[Commands.swift](Sources/ReolinkAPI/Commands/Commands.swift) for examples.

## Privacy

Reolens runs entirely on your devices. It talks only to:

- Your Reolink cameras, NVRs, and Home Hubs (over the local network)
- iCloud Drive (for syncing the camera list across your own Apple
  devices ‚Äî opt-out via System Settings ‚Üí iCloud)
- `reolens.io/appcast.xml` (macOS only, for Sparkle update checks;
  disable in Settings)
- Apple's TestFlight / App Store (iOS, for app updates)

That's the entire network surface. No third-party analytics. No remote
crash reporting. No telemetry. No accounts.

**Credentials are device-local.** Camera passwords live in each device's
Keychain (`kSecAttrSynchronizable: false` ‚Äî explicitly never synced),
and the iCloud Drive sync is metadata-only (display name, host, port,
username, grid layout). See [`AGENTS.md`](AGENTS.md) ¬ß3, ¬ß4 for the
full credential model.

**Screenshots in marketing materials use procedurally-rendered camera
scenes**, not real footage. Generate / regenerate them with
[`Scripts/make-stock-camera-views.swift`](Scripts/make-stock-camera-views.swift) and
[`Scripts/make-placeholder-screenshots.swift`](Scripts/make-placeholder-screenshots.swift).
